background

What am I doing?

Found Eats is an online marketplace for specialty foods where users can browse the latest products, discover the newest food makers, and purchase their favorite artisanal foods. Food makers looking to sell their products are directed to a vendor-facing website where they can set-up their company to become an active vendor on the online marketplace. After the vendor set-up is complete, this website will be used to manage incoming orders, update product or company details and coordinate product shipping and payment. This project is an optimized redesign of the current Found Eats Vendor Account website.

ux research

Understanding the User

The first step was getting to know the vendors that would be using the site. By talking to them, and walking them through the current website, I familiarized myself with their stories, daily routines, and the role that they hoped Found Eats would play in the success of their businesses.

Use Cases

My redesign centers mostly around fixing the problems created by the current design in Use Case #1. As a result, Use Case #2 and Use Case #3 are enhanced.

  • 1

    I am a new vendor who is going through the set-up process the first time.

  • 2

    I am a vendor who is already up and live on the site, and am visiting the site to modify my information or products.

  • 3

    I am a live vendor checking my account to maintain the orders that come in through the site.

Analyzing Problems with the Current Site

  1. Vendors are ignoring mandatory sections because they don't know what is required to complete set-up.
  2. Vendors leave fields blank or incomplete because they don't understand what the form is asking.
  3. Vendors are becoming sloppy with their product and company descriptions as they rush through it due to how long and involved it seems.
  4. Vendors are not completing the sign up process due to frustration with the process and waning interest.

strategy

My Vision

  • Make the user feel confident by creating a short, clear path to completion
  • Make the process seem easy with an intuitive layout, and accessible explanations that rely largely on visual content.
  • Make the experience exciting using aesthetic appeal and keeping the end goal in mind.

Accomplishing these goals will transform what is currently a daunting but necessary chore into an opportunity for the vendors to share their stories, show their passion, and communicate with a large community of foodlovers. For Found Eats, this new design will require minimal maintenance, reduce the amount of errors and increase the rate by which new vendors successfully complete set-up.

My Game Plan

  • 1
    Condense the Site Map
    • Reduce the number of sections
    • Reduce the number of clickaway pages
  • 2
    Shepherd the User through Set-up
    • Limit access to non-active sections
    • Implement a Pop-up Tutorial to walk the user through the process
  • 3
    Focus and Orient the User
    • Introduce global navigation and progress meter
    • Introduce a section-wide checklist and local navigation element
  • 4
    Mirror the Found Eats Marketplace
    • Create a look and feel similar to the customer site
    • Use in-place editing to excite the user and clarify information
  • 5
    Make Information Accessible
    • Display relevent information
    • Use pictoral explanations
    • Implement a fixed FAQ panel

solution #2

Shepherd the User through Set-up

removing the dashboard

The current landing page gives the user access to all sections of the website, including those that are not relevent to vendors who have not completed the set-up process. This situation creates an ambiguous path , creating confusion and frustration. The new design doesn’t introduce a dashboard until the user has completed the set-up process.

current landing page: dashboard

pop-up walkthrough

Pop-ups guide the user through each step of the set-up process. This shepherds the user step-by-step from start to finish. Within sections, pop-ups guide the user and provide extra explanatory information about the input fields.

new landing page (wireframe)

Solution #2 Summary

Implementing a walk-through that leads the user to completion in contrast to having a single page which serves as the sole compass for all use cases allows the user to stay focused on one step at a time, which makes using the website less overwhelming. It highly reduces the instance of errors including ignoring mandatory sections or individual input fields and greatly increases the chance of a successful vendor set-up.

solution #3

Focus and Orient the User

combining navigation and progress

In the new design, the progress bar doubles as tabs for global navigation. This organizes the sections of the website by their respective step in the set-up process, giving the user both direction and purpose.

current design : nav bar
current design: progress bar

combining navigation and progress

In the new design, the progress bar doubles as tabs for global navigation. This organizes the sections of the website by their respective step in the set-up process, giving the user both direction and purpose.

progress bar as a recurring feature

The nav/progress bar appears on every page with a constant reminder of how many more steps must be completed. Fewer choices and the absence of a dashboard creates a clear trajectory to focus the user.

new design: nav/progress bar design

local checklist

The new design features a checklist of all the necessary fields that must be completed in order for a given section to be finished. Once a field is completed, a checkmark will appear next to its respective list item. If clicked, a list item will navigate the user to its respective entry field. This prevents the user from overlooking or intentionally skipping any fields.

Solution #3 Summary

Having both a global and local progress meter/navigation system negates the possibility of vendors accidentally or deliberately leaving entire sections or individual input fields incomplete. It succinctly answers what to fill out, and where to find it.

Solution #4 Summary

Mirroring the customer-facing marketplace not only improves clarity, and communicates significance to the user, but also creates a familiar, cohesive look. Enhancing the aesthetic appeal of the website alone will make the user more excited to complete their profile. Often the company stories include highly personal motivations like passing down grandma’s recipe or coming up with creations for a family member with dietary restrictions. My optimized Vendor Account Website is designed to not only ellicit strong emotions like nostalgia, warmth or pride, but also provide a means to express them by uploading pictures and sharing experiences on their company profile.

solution #5

Make Information Accessible

un-hide information

The current design displays information when the user hovers over the question mark icon next to its respective input field. Users enter wrong information because they are not accessing the explanations. The new design pulls the information out of the hover state, and displays them directly on the page all the time.

infographics

Instead of relying solely on text like the current design, the new design makes use of pictures to send clearer messages that people will be more likely to read.

current design: hover explanation
new design: interactive infographic

field-specific FAQs

When the user clicks on an input field, FAQs specific to the selected field pop up on the right-panel. When the user clicks out of that field those FAQs dissapear. Different fields feature different FAQs.

new design (wireframe)

field-specific FAQs

When the user clicks on an input field, FAQs specific to the selected field pop up on the right-panel. When the user clicks out of that field those FAQs dissapear. Different fields feature different FAQs.

Solution #5 Summary

In order to make information more accessible I decided to replace a visually short page for a longer one where the user has to scroll through infographics and more explanations dropped right on the page. The design will force the user to at least glance through explanations when filling out input fields, and guarentees that the user will be able to access them if they have questions. The result will be fewer mistakes, less frustration and a higher completion rate.
